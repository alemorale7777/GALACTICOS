1. FULL SCREEN MAP — HIGHEST PRIORITY
The game board must fill the entire phone screen edge to edge.

- Canvas width: window.innerWidth (no margins, no padding)
- Canvas height: window.innerHeight minus the UI bar height at bottom
- On resize / orientation change: recalculate and redraw
- Node positions: scale proportionally to new canvas size
  (multiply all x,y coordinates by scaleX and scaleY factors)
- Ensure no nodes are cut off at edges — add 5% padding from edges minimum
- The UI bar (Army % buttons + War Cry) stays pinned to bottom
- Remove any fixed pixel dimensions on the canvas element

---

2. NODE INFLUENCE CIRCLES — FIX OVERLAP
The soft glow circles around nodes are too large and cluttering the map.

- Reduce influence circle radius by 35%
- Change influence circle opacity from current to max 0.15
- Use a tighter radial gradient: full color at node edge, 
  fully transparent at influence circle edge
- Selected node ring: keep it, but make it crisp (2px stroke, 
  not a filled circle)
- This will make the map feel open and readable

3. UNIT SWARM CLEANUP
Units orbiting/stacking on a castle look like a chaotic blob.

- Units waiting at a castle should NOT visually orbit or cluster
- In-transit units only: show trail and glow
- Cap visible units per fleet at 12 (if fleet > 12 units, 
  show 12 dots + a small number badge like "+8" floating above)
- Spread in-transit units in a loose V-formation or 
  staggered column (not a single point)
- Unit radius: reduce by 1px to declutter
